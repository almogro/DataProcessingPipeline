System Overview

Purpose

The Functional Data Processing Pipeline is a Scala-based application that processes movie data using Apache Spark, demonstrating advanced functional programming principles including immutability, pure functions, higher-order functions, currying, and combinators.

Key Features

  • Data Ingestion: CSV file parsing and validation
  • Data Processing: Functional transformations and enrichments
  • Data Output: Structured CSV output with enhanced metadata
  • Quality Filtering: High-quality movie selection criteria
  • Analytics: Genre statistics and decade-based analysis

Detailed Transformation Analysis

This section provides a comprehensive explanation of how every transformation is expressed in the code, with detailed block-by-block analysis.

1. Data Parsing Transformations

CSV String → MovieRecord

// In DataParsingFunctions.scala
def parseMovieRecord(line: String): Option[MovieRecord] = {
  val fields = line.split(",")  // Split CSV line by comma into array of strings
  
  if (fields.length >= 6) {     // Check if we have at least 6 fields (id, title, year, genre, rating, votes)
    try {
      Some(MovieRecord(         // Create Some wrapper around MovieRecord for success case
        id = fields(0).trim,    // Extract first field, remove whitespace
        title = fields(1).trim, // Extract second field, remove whitespace
        year = fields(2).trim.toInt,    // Extract third field, convert string to integer
        genre = fields(3).trim, // Extract fourth field, remove whitespace
        rating = fields(4).trim.toDouble, // Extract fifth field, convert string to double
        votes = fields(5).trim.toLong     // Extract sixth field, convert string to long
      ))
    } catch {
      case _: NumberFormatException => None    // If number conversion fails, return None
      case _: IndexOutOfBoundsException => None // If array access fails, return None
    }
  } else {
    None  // If not enough fields, return None
  }
}

What's happening:

  • Input: Raw CSV string like "1,The Shawshank Redemption,1994,Drama,9.3,2500000"
  • Process: Split by comma, validate field count, convert types safely
  • Output: Some(MovieRecord(...)) on success, None on failure
  • Error Handling: Uses Option type to avoid exceptions

How it's used in pipeline:

// In DataPipeline.scala
val movieRecords = rawData
  .map(DataParsingFunctions.parseMovieRecord)  // Apply parsing function to each CSV line
  .filter(_.isDefined)                         // Keep only successful parses (Some values)
  .map(_.get)                                  // Extract MovieRecord from Some wrapper

What's happening in each step:

  1. .map(parseMovieRecord): Transforms Dataset[String] → Dataset[Option[MovieRecord]]
  2. .filter(_.isDefined): Removes None values, keeps only Some values
  3. .map(_.get): Unwraps Some(MovieRecord) → MovieRecord

2. Data Filtering Transformations

Quality Filtering

// In DataFilteringFunctions.scala
def isHighQualityMovie(movie: MovieRecord): Boolean = {
  movie.rating >= 7.0 && movie.votes >= 1000  // Return true if both conditions are met
}

What's happening:

  • Input: MovieRecord object
  • Process: Check if rating ≥ 7.0 AND votes ≥ 1000
  • Output: true if movie is high quality, false otherwise
  • Pure Function: No side effects, same input always produces same output

Curried Filtering Functions

// In DataFilteringFunctions.scala
def isFromDecade(decade: String): MovieRecord => Boolean = {
  val decadeStart = decade.take(4).toInt  // Extract first 4 characters and convert to int
  val decadeEnd = decadeStart + 9         // Calculate decade end year
  
  (movie: MovieRecord) => movie.year >= decadeStart && movie.year <= decadeEnd  // Return function
}

What's happening:

  • Input: decade string like "1990s"
  • Process:
    1. Extract "1990" from "1990s" using take(4)
    2. Convert to integer: 1990
    3. Calculate end year: 1990 + 9 = 1999
    4. Return a function that checks if movie year is between 1990-1999
  • Output: Function MovieRecord => Boolean
  • Currying: Function that takes one parameter and returns another function

3. Data Enrichment Transformations

Main Enrichment Pipeline

// In DataTransformationFunctions.scala
def enrichMovieRecord(movie: MovieRecord): ProcessedMovieRecord = {
  ProcessedMovieRecord(
    movie = movie,                                    // Keep original movie data
    decade = computeDecade(movie.year),              // Add computed decade
    ratingCategory = categorizeRating(movie.rating), // Add rating category
    popularityScore = computePopularityScore(movie.rating, movie.votes)  // Add popularity
  )
}

What's happening:

  • Input: MovieRecord with basic fields
  • Process: Create new ProcessedMovieRecord with additional computed fields
  • Output: Enriched record with decade, category, and popularity score
  • Immutability: Creates new object instead of modifying existing one

Pattern Matching for Rating Categories

private def categorizeRating(rating: Double): String = {
  rating match {                    // Pattern matching expression
    case r if r >= 9.0 => "Excellent"  // If rating >= 9.0, return "Excellent"
    case r if r >= 7.0 => "Good"       // If rating >= 7.0, return "Good"
    case r if r >= 5.0 => "Average"    // If rating >= 5.0, return "Average"
    case _ => "Poor"               // For any other value, return "Poor"
  }
}

What's happening:

  • Input: rating as Double (e.g., 9.3)
  • Process:
    1. Check if rating ≥ 9.0 → "Excellent"
    2. If not, check if rating ≥ 7.0 → "Good"
    3. If not, check if rating ≥ 5.0 → "Average"
    4. Otherwise → "Poor"
  • Output: String category
  • Pattern Matching: Uses match expression with guard conditions

4. Spark Transformations

Complete Pipeline Expression

// In DataPipeline.scala - The complete transformation pipeline
def processData(rawData: Dataset[String]): Dataset[ProcessedMovieRecord] = {
  // Step 1: Parse CSV lines into MovieRecords
  val movieRecords = rawData
    .map(DataParsingFunctions.parseMovieRecord)  // String → Option[MovieRecord]
    .filter(_.isDefined)                         // Keep only Some values
    .map(_.get)                                  // Extract MovieRecord from Option
  
  // Step 2: Filter high-quality movies
  val filteredMovies = movieRecords
    .filter(DataFilteringFunctions.isHighQualityMovie(_))  // Keep only quality movies
  
  // Step 3: Enrich with additional data
  val processedMovies = filteredMovies
    .map(DataTransformationFunctions.enrichMovieRecord)  // Add decade, category, score
  
  processedMovies  // Return enriched dataset
}

What's happening in each step:

Step 1 - Parsing:

  1. rawData: Dataset[String] (raw CSV lines)
  2. .map(parseMovieRecord): Transform each string to Option[MovieRecord]
  3. .filter(_.isDefined): Remove None values, keep only Some values
  4. .map(_.get): Unwrap Some(MovieRecord) to MovieRecord
  5. Result: Dataset[MovieRecord] with parsed data

Step 2 - Filtering:

  1. movieRecords: Dataset[MovieRecord] from step 1
  2. .filter(isHighQualityMovie(_)): Apply quality filter to each movie
  3. Result: Dataset[MovieRecord] with only high-quality movies

Step 3 - Enrichment:

  1. filteredMovies: Dataset[MovieRecord] from step 2
  2. .map(enrichMovieRecord): Add decade, rating category, popularity score
  3. Result: Dataset[ProcessedMovieRecord] with enriched data

Key Expression Patterns

  1. Higher-Order Functions: Functions that take other functions as parameters
    • Example: .map(f) where f is a function parameter
  2. Function Composition: Chaining functions together
    • Example: g(f(value)) or using combinators
  3. Pattern Matching: Using match expressions for conditional logic
    • Example: rating match { case r if r >= 9.0 => "Excellent" }
  4. Immutable Data: Creating new objects instead of modifying existing ones
    • Example: ProcessedMovieRecord(movie = movie, decade = computeDecade(...))
  5. Error Handling: Using Option and Either for safe operations
    • Example: Option[MovieRecord] for parsing, Either[Throwable, B] for retry
  6. Tail Recursion: Optimized recursive functions
    • Example: computeAverageRatingTail with accumulator parameters
  7. Currying: Functions that return other functions
    • Example: isFromDecade(decade: String): MovieRecord => Boolean
  8. Lambda Expressions: Anonymous functions
    • Example: _.isDefined is shorthand for x => x.isDefined

Each transformation is expressed as a pure function that takes input and produces output without side effects, making the entire pipeline predictable and testable! 🚀
